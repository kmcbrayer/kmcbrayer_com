Regustrator

Tech: Angular, THREE.js, Grunt.js, Yeoman with angular-generator

The project was an idea I had to use Angular directives to place three dimensional objects on a webpage with just HTML and a little JSON. It uses THREE.js to do the actual renderin. The purpose of the project was to abstract away the render loop elements, object creation, and setup code and put them in Angular directives. So instead of 50 or so lines of JavaScript you would end up with: 

    <div class="window" ng-controller="MainCtrl">

      <screen id="main_screen"></screen>
      <camera></camera>

      <box props="box2"></box>
      <box options="{'x':3,'y':2,'z':3,'color':100}"></box>

      <line props="line1"></line>
      <line options="{'x1':0,'y1':-10,'x2':0,'y2':10}"></line>
      <line options="{'x1':0,'z1':-10,'x2':0,'z2':10}"></line>

      <triangle></triangle>
      <plane></plane>

      <text>This is text</text>

      <renderer></renderer>
    </div>

So any object elements between "camera" and "renderer" get rendered on the page. The attributes "props" and "options" roughly translate to external and inline styling respectivly. Props looks inside an angular service for a predefined json object with the matching name e.g. "box2". Options is just a stringified json object.

Feeder

Tech: Node, Express, Angular, karma-jasmine-phantomJS, Grunt, and node-passport with social media APIs like FaceBook, Twitter, Google/Youtube, and Instagram.

The main purpose of this project is to consolidate several social media feeds in chronological order. This is my first real go at a Node server and I found that all of the npm passport modules are pretty amazing. Logging is done with Winston JS . Grunt and all of its plugins are great for tests and the live reload on save. This was my first expirence with this an I will never go back.
