Regustrator

Tech: Angular, THREE.js, Grunt.js, Yeoman with angular-generator

The project was an idea I had to use Angular directives to place three dimensional objects on a webpage with just HTML and a little JSON. I have some expierence with XNA and rendering 3D objects from college courses. So when I found THREE.js I got really excited. The purpose of the project was to abstract away the render loop elements, object creation, and setup code Angular directives. So instead of 50 or so lines of JavaScript you would end up with: 

    <div class="window" ng-controller="MainCtrl">

      <screen id="main_screen"></screen>
      <camera></camera>

      <box props="box2"></box>
      <box options="{'x':3,'y':2,'z':3,'color':100}"></box>

      <line props="line1"></line>
      <line options="{'x1':0,'y1':-10,'x2':0,'y2':10}"></line>
      <line options="{'x1':0,'z1':-10,'x2':0,'z2':10}"></line>

      <triangle></triangle>
      <plane></plane>

      <text>This is text</text>

      <renderer></renderer>
    </div>

So any object elements between "camera" and "renderer" get rendered on the page. The attributes "props" and "options" are the little JSON I mentioned earlier. Those two attributes roughly translate to external and inline styling respectivly. Props looks inside an angular service for a predefined json object with the matching name e.g. "box2". Options, as you can see, is just a stringified json object.

Feeder

Tech: Node, Express, Angular, and a testing suite consisting of karma/jasmine with phantomJS. Lots of node-passport with social media APIs like FaceBook, Twitter, Google/Youtube, and Instagram.

warning: still a very rough project on the front end

The main purpose of this project is to consolidate several social media feeds. And in the future maybe news RSS feeds. All in chronological order. A really interesting problem I've run into is that the feeds typically are not integrated very uniformly. By nature, twitter will have more entries per hour than youtube video uploads. So I am looking forward to putting an algorithm in place to properly mix up and sort these feeds. I'm thinking a priority based system. Picture all those standardized tests where you know the next answer cant be C because the previous, like, 3 were so you pick B instead. I don't know yet. But I'm excited about it and am saving it for last, kind of a treat for finishing writing all the tests and directives.

Something in this project I'm very proud of, and the single task that has taken the most time, is that I restructured a yeoman angular-fullstack app to comply with Angulars new best practices. It took alot of tinkering with the Gruntfile and bower_components location ( recursive inclusion while running karma tests) but I'm very happy with this new application organization.
